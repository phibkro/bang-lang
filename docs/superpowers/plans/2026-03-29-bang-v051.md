## Task 2: `on` Returns Subscription with `.abort`

**Files:**
- Modify: `packages/core/src/Interpreter.ts`
- Modify: `packages/compiler/test/On.test.ts`

- [ ] **Step 1: Wrap subscribers with active flag**

In Interpreter.ts, change subscriber registration. Instead of pushing a raw function, push an object with an `active` flag:

```typescript
interface Subscriber {
  active: boolean;
  fn: (newValue: Value) => Effect.Effect<void, EvalError>;
}
```

In the `<-` handler, before firing each subscriber, check `sub.active`:
```typescript
for (const sub of cell.value.ref.subscribers) {
  if (sub.active) yield* sub.fn(newValue);
}
```

Update `MutCell.ref.subscribers` type from `Array<(newValue: Value) => ...>` to `Array<Subscriber>` in Value.ts.

- [ ] **Step 2: Return Subscription with abort**

In the `OnExpr` handler, after registering the subscriber:
```typescript
const subscriber: Subscriber = { active: true, fn: sub };
sourceCell.ref.subscribers.push(subscriber);

// Build abort: sets active = false
return Tagged({
  tag: "Subscription",
  fields: [/* abort thunk — we need a way to call subscriber.active = false */],
  fieldNames: ["abort"],
});
```

For the abort thunk: since we can't build a synthetic Closure that sets a JS flag, use a simpler approach. Add a `BuiltinFn` variant to Value, or represent abort as a Tagged value that the Force handler recognizes.

**Simplest approach:** Store the subscriber reference directly in the Tagged value. When `.abort` is accessed and forced, set `subscriber.active = false`. Implement this in the DotAccess handler:

```typescript
// In DotAccess, for Subscription.abort:
if (obj._tag === "Tagged" && obj.tag === "Subscription" && e.field === "abort") {
  // The subscriber ref is stored... we need it accessible
}
```

**Practical approach:** Store a mutable abort flag object as the first field of Subscription. The abort field is a MutCell wrapping a Bool. Force `.abort` sets it to false. The subscriber checks it.

Actually simplest: just use the `active` flag on the subscriber object. Store the subscriber object reference in the Subscription's fields as an opaque value. Add the abort behavior directly in DotAccess + Force.

**Recommended implementation:**
1. Keep the subscriber `{ active: true, fn }` pattern
2. Add a special Value variant or use existing Tagged: `Tagged({ tag: "Subscription", fields: [], fieldNames: [] })` with a captured JS closure reference stored out-of-band
3. When `.abort` is accessed on a Subscription, return a synthetic thunk that deactivates
4. In practice: use a WeakMap or closure to associate Subscription identity with the subscriber

**Simplest viable:** Don't return a real closure. Instead, when `!sub.abort` is forced, the Force handler checks if `sub` is a Subscription-tagged value and deactivates. Store the subscriber array + index in the Tagged value's fields as numeric values.

Let the implementer choose the simplest working approach. The tests define the contract.

## Task 5: `use` Resource Protocol (CPS)

**Files:**
- Modify: `packages/core/src/Interpreter.ts`
- Modify: `packages/compiler/test/Use.test.ts`

- [ ] **Step 1: Implement CPS in Block handler**

In Interpreter.ts, replace the sequential `for` loop in the `Block` match arm with an index-based approach that detects `!use`:

```typescript
Match.tag("Block", (e) =>
  Effect.gen(function* () {
    let blockEnv = env;
    for (let i = 0; i < e.statements.length; i++) {
      const stmt = e.statements[i];
      // Detect !use pattern: ForceStatement(Force(UseExpr(...)))
      if (stmt._tag === "ForceStatement" &&
          stmt.expr._tag === "Force" &&
          stmt.expr.expr._tag === "UseExpr") {
        const useExpr = stmt.expr.expr;
        const provider = yield* evalExpr(useExpr.value, blockEnv);
        // Build continuation from remaining stmts + return expr
        const remaining = e.statements.slice(i + 1);
        const contBlock = new Ast.Block({
          statements: [...remaining],
          expr: e.expr,
          span: e.span,
        });
        const contLambda = new Ast.Lambda({
          params: [useExpr.name],
          body: contBlock,
          span: e.span,
        });
        const callback = yield* evalExpr(contLambda, blockEnv);
        // Apply provider to callback
        if (provider._tag === "Closure" || provider._tag === "Constructor") {
          return yield* applyValue(provider, [callback], e.span);
        }
        // Fallback: provider is a plain value, bind directly
        blockEnv = HashMap.set(blockEnv, useExpr.name, provider);
        continue;
      }
      blockEnv = yield* evalStmt(stmt, blockEnv);
    }
    return yield* evalExpr(e.expr, blockEnv);
  }),
),
```

Import `Ast` at the top (may need to change `import type` to `import`).

## Task 6: Nested Pattern Codegen

**Files:**
- Modify: `packages/compiler/src/Codegen.ts`
- Modify: `packages/compiler/test/NestedPatterns.test.ts`

- [ ] **Step 1: Group arms by outer constructor tag**

In Codegen.ts `emitExpr` MatchExpr handler, before emitting arms:

1. Group constructor-pattern arms by their outer tag
2. For each group with >1 arm or nested sub-patterns, emit a single `Match.tag` with inner `Match.value`
3. Wildcard/binding/literal arms go at the end as `Match.orElse`

```typescript
// Group arms by outer tag
const groups = new Map<string, Ast.Arm[]>();
const otherArms: Ast.Arm[] = [];
for (const arm of e.arms) {
  if (arm.pattern._tag === "ConstructorPattern") {
    const tag = arm.pattern.tag;
    if (!groups.has(tag)) groups.set(tag, []);
    groups.get(tag)!.push(arm);
  } else {
    otherArms.push(arm);
  }
}

// Emit grouped Match.tag calls
const tagCalls: string[] = [];
for (const [tag, arms] of groups) {
  if (arms.length === 1 && !hasNestedConstructorPattern(arms[0].pattern)) {
    // Simple: single arm, flat destructuring
    tagCalls.push(emitSimpleMatchTag(tag, arms[0], decls, mutNames, hoisted));
  } else {
    // Complex: multiple arms or nested patterns → inner Match.value
    tagCalls.push(emitGroupedMatchTag(tag, arms, decls, mutNames, hoisted));
  }
}
```

Implement `emitGroupedMatchTag` to emit nested `Match.value(...).pipe(...)`.

- [ ] **Step 1: Detect braced handler syntax in parser**

In `parseApplication` or `parseDotAccess`, after parsing `expr.handle` or `expr.catch`, check if the next argument starts with `{`. If so, parse as handler arms (TypeIdent `->` Expr, separated by `,`) and desugar into chained DotAccess + App.

The detection point: in `parseApplication`, when collecting arguments for a DotAccess function, if the DotAccess field is `"handle"` or `"catch"` and the next token is `{`:

```typescript
// In parseApplication, after detecting func is DotAccess with "handle"/"catch":
if (e.func._tag === "DotAccess" &&
    (e.func.field === "handle" || e.func.field === "catch")) {
  const isBrace = yield* check(s, "Delimiter", "{");
  if (isBrace) {
    return yield* parseBracedHandler(e.func.object, e.func.field, s);
  }
}
```

Implement `parseBracedHandler`:
```typescript
const parseBracedHandler = (obj: Ast.Expr, method: string, s: ParseState): P<Ast.Expr> =>
  Effect.gen(function* () {
    const [, s1] = yield* expect(s, "Delimiter", "{");
    // Parse TypeIdent -> Expr pairs separated by ,
    const pairs: Array<{ typeName: Ast.Expr; impl: Ast.Expr }> = [];
    // ... parse pairs ...
    const [, sEnd] = yield* expect(sN, "Delimiter", "}");
    // Desugar: chain .handle calls
    let result = obj;
    for (const pair of pairs) {
      result = new Ast.App({
        func: new Ast.DotAccess({ object: result, field: method, span }),
        args: [pair.typeName, pair.impl],
        span,
      });
    }
    return [result, sEnd] as const;
  });
```
